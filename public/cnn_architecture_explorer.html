T-01 He et al., CVPR 2016
Deep Residual Learning for Image Recognition

Before ResNet, the assumption was simple: more layers = better accuracy. In 2015, He et al. found the opposite. A 56-layer plain network performed worse than a 20-layer one — not just on test data, but on training data too. This ruled out overfitting. The problem was optimization.

The Degradation Problem

  • Gradients shrink exponentially as they backpropagate through many layers (vanishing gradients)
  • The error signal effectively disappears before reaching shallow layers
  • Networks struggle to learn identity mappings — even when "do nothing" is the optimal solution
  • Batch normalization helped but didn't fully solve the problem beyond ~30 layers
Core insight: If the optimal transformation is close to identity, it's easier to learn F(x) = 0 (a small residual/deviation) than to learn H(x) = x directly. The skip connection handles the identity for free — the layers only learn the correction.

Prior Work Limitations

  • Highway Networks (2015): Used gated skip connections, but gates required extra parameters and the depth gains were modest
  • Plain deep nets (VGG etc.): Accuracy saturated and then degraded with depth
  • Batch Normalization: Fixed internal covariate shift but did not solve the fundamental gradient flow problem at extreme depth

A residual block adds the input x directly to the output of a small stack of layers F(x). The final output H(x) = F(x) + x. The layers no longer need to learn the full mapping — only the residual (the deviation from the identity).

H(x) = F(x) + x F(x) = W₂ · ReLU(W₁x + b₁) + b₂ ← the residual mapping If optimal F(x) ≈ 0 → H(x) ≈ x (identity, effortlessly) If optimal F(x) ≈ Δ → the block learns only the small correction Δ

Key Design Choices

  • Shortcut with no parameters: When input/output dimensions match, the skip is a direct wire — zero extra cost
  • Projection shortcut: When dimensions differ (different channels or stride), a 1×1 conv maps x to the right size
  • BN before ReLU: Normalizes before activation — stabilizes training at all depths
  • Global Average Pooling: Replaces large fully-connected layers at the end — huge parameter reduction
ResNet-50 uses the bottleneck design: 256 channels → 64 (1×1) → 64 (3×3) → 256 (1×1). This 3-layer bottleneck costs the same FLOPs as a single 3×3 conv on 64 channels, but has 256-channel capacity.

Influence on Subsequent Work

  • Became the universal backbone for Faster R-CNN (detection), Mask R-CNN (segmentation), FPN (feature pyramids)
  • First architecture proven to train networks of 1000+ layers reliably
  • Residual connections are now in Transformers (each attention block has a skip), LSTMs, and U-Nets
  • As of 2024, ResNet variants are still deployed in production systems worldwide

ResNet fundamentally changed how depth is perceived in neural networks — from an obstacle into a design dial. The residual learning hypothesis proved networks prefer to learn small corrections rather than complete transformations from scratch.

Limitations

  • Skip connections require two tensors in memory simultaneously — higher memory footprint than plain nets
  • Standard 3×3 kernels limit how fast the receptive field grows per layer
  • No channel-wise attention — all channels treated equally regardless of input content
  • FLOPs grow quadratically with width — less efficient than depthwise approaches
Open question: Why exactly does residual learning work? Current theories include: (1) it creates an implicit ensemble of shallow networks, (2) it smooths the loss landscape, (3) it creates gradient highways. No single complete theory exists.

What We Can Learn

  • Reparameterize difficult optimization problems — learn deviations rather than full targets
  • Simple inductive biases (identity shortcut) can have outsized impact on trainability
  • Architecture choices are not separable from optimization dynamics

DenseNet
T-02 Huang et al., CVPR 2017
Densely Connected Convolutional Networks

ResNet showed skip connections help. But it only connected a layer to the one immediately after it. DenseNet asks: why stop there? What if every layer connects directly to every subsequent layer?

  • Even with ResNet, early-layer features can be diluted or overwritten by later layers
  • Feature maps from shallow layers (edges, textures) are still useful deep in the network
  • ResNet adds F(x)+x — the skip blends features. DenseNet concatenates — all features are preserved
  • Implicit deep supervision: gradients flow directly from the loss to every layer
Key insight: Concatenating (not adding) all previous feature maps forces each new layer to learn complementary, non-redundant features. This leads to dramatically better parameter efficiency.

In a dense block with L layers, there are L(L+1)/2 connections. Layer l receives the concatenation of all previous outputs [x₀, x₁, …, x_{l-1}] as input. Each layer produces exactly k new feature maps — the growth rate.

x_l = H_l([x_0, x_1, ..., x_{l-1}]) Input channels to layer l: k₀ + k × (l-1) k₀ = initial channels, k = growth rate H_l = BN → ReLU → Conv 3×3 (composite function) Bottleneck variant: H_l = BN → ReLU → Conv 1×1 (4k ch) → BN → ReLU → Conv 3×3 (k ch)

Key Design Choices

  • Growth rate k: Typically 12–32. Each layer adds only k channels — keeping the network narrow
  • Transition layers: Between dense blocks — 1×1 conv (compression) + 2×2 avg pool. Compression ratio θ=0.5 halves channels
  • Bottleneck: 1×1 conv before 3×3 reduces the concatenated input width cheaply
  • No width explosion: With k=32 and 4 layers, layer 4 input = k₀ + 3×32 channels — manageable
With k=12, a 4-layer dense block: Layer 1 gets 16ch → outputs 12. Layer 2 gets 28ch → outputs 12. Layer 3 gets 40ch → outputs 12. Layer 4 gets 52ch → outputs 12. Total connections: 4×5/2 = 10 vs 4 in a plain net.

  • Implementation: Concatenation across layers requires careful memory management (memory-efficient DenseNet paper addressed this)
  • Hardware efficiency: Wide concatenated tensors don't map as cleanly to GPU tensor cores as standard convolutions
DenseNet's real lasting contribution may be demonstrating implicit deep supervision — the idea that every layer should receive a direct gradient signal from the loss. This influenced training strategies more than the exact connectivity pattern.

MobileNet
T-03 Howard et al., 2017
Efficient Convolutional Neural Networks for Mobile Applications

AlexNet required 240MB of weights and ~720M multiply-accumulate operations per inference pass — far too heavy for a smartphone, drone, or IoT sensor. MobileNet asks: can we factor the convolution operation into something much cheaper without losing accuracy?

  • Mobile/embedded: <512MB RAM, no discrete GPU, strict power budget
  • A standard 3×3 conv on 128 input channels to 256 output channels = 3×3×128×256 = 294,912 parameters per layer
  • SqueezeNet reduced parameters, but not FLOPs — inference speed barely improved
Mathematical insight: Standard convolution mixes spatial filtering (where) and channel combination (what) in a single step. These two operations can be factored — spatial first (depthwise, one filter per channel), then channel (pointwise, 1×1) — with a theoretically provable reduction of 1/N + 1/D_k² ≈ 8–9× for k=3.
Standard Conv: params = D_k × D_k × M × N FLOPs = D_k × D_k × M × N × D_f × D_f Depthwise Separable Conv: Depthwise: D_k × D_k × M (1 filter per input channel) Pointwise: 1 × 1 × M × N (combine channels) Total FLOPs = D_k²·M·D_f² + M·N·D_f² Reduction ratio: 1/N + 1/D_k² For N=256, D_k=3: 1/256 + 1/9 ≈ 1/8.6 → 8.6× fewer computations

Scaling Multipliers

  • Width multiplier α ∈ (0,1]: Uniformly thins all layers — α=0.75 gives 75% channels everywhere. FLOPs reduce by α²
  • Resolution multiplier ρ: Reduces input image resolution (224→192→160→128). FLOPs reduce by ρ²
  • Combined: α=0.5, ρ=0.5 → 4× fewer parameters AND 4× fewer FLOPs → 16× total reduction
  • ReLU6: Clips activation at 6, enabling fixed-point (int8) quantization without saturation issues

SqueezeNet
T-04 Iandola et al., 2016
AlexNet-level accuracy with 50× fewer parameters

AlexNet won ImageNet 2012 with 240MB of weights. Distributing, storing, and updating models this large is impractical — for embedded deployment, OTA model updates, or FPGA deployment. SqueezeNet's explicit goal: AlexNet-level accuracy in under 1MB, designed small from scratch.

Three Design Strategies

  • Strategy 1: Replace 3×3 filters with 1×1 filters wherever possible (9× fewer parameters per filter)
  • Strategy 2: Reduce the number of input channels to 3×3 filters using a "squeeze" layer before them
  • Strategy 3: Downsample late — keep large spatial feature maps longer to preserve information
All three strategies reduce parameters without reducing accuracy. Strategy 1+2 directly reduce parameter count. Strategy 3 compensates for the lost expressiveness by keeping feature maps large longer.

The Fire module is SqueezeNet's core. It has two stages: squeeze (1×1 only) then expand (parallel 1×1 and 3×3). The squeeze reduces channels cheaply before the expensive 3×3 expand operation.

Fire Module: Input: M channels → Squeeze: 1×1 conv → s₁ channels (s₁ < M, typically s₁=16) → Expand A: 1×1 conv → e₁ channels ↗ in parallel → Expand B: 3×3 conv → e₃ channels ↘ concat → Output: e₁ + e₃ channels (typically e₁=e₃=64 → 128ch out) Parameter count: Squeeze: M × s₁ Expand A: s₁ × e₁ Expand B: s₁ × 9 × e₃ Total: M·s₁ + s₁·e₁ + 9·s₁·e₃

Full Architecture

  • conv1 (96 filters, 7×7, stride 2) → maxpool → 8× Fire modules → conv10 (1000 filters, 1×1) → avg pool → softmax
  • No fully-connected layers — FC layers are traditionally 90% of VGG's parameters
  • SqueezeNet + Deep Compression (Han et al.) → 0.47MB (510× smaller than AlexNet)

EfficientNet
T-05 Tan & Le, ICML 2019
Rethinking Model Scaling for Convolutional Neural Networks

When you want a more powerful CNN, how do you grow it? Previous work scaled one dimension at a time: ResNet goes deeper, WideResNet goes wider, higher-resolution inputs add resolution. But these dimensions interact — more depth only helps if the resolution is high enough to provide fine-grained spatial information, and width only helps if there are enough layers to combine the extra channels.

Key question: Is there a principled way to scale all three dimensions (depth, width, resolution) simultaneously? EfficientNet's answer: yes — balance them with a single compound coefficient φ, grid-searched under a fixed FLOP constraint.
Compound scaling: Depth: d = α^φ (α = 1.2) Width: w = β^φ (β = 1.1) Resolution: r = γ^φ (γ = 1.15) Constraint: α · β² · γ² ≈ 2 (doubling φ by 1 ≈ doubles FLOPs) EfficientNet-B0 (φ=1): 224px, 5.3M params, 77.1% top-1 EfficientNet-B7 (φ=8): 600px, 66M params, 84.3% top-1

Baseline (B0) Architecture

  • Found via NAS — uses MBConv blocks (inverted residual from MobileNetV2)
  • Squeeze-and-Excitation (SE) blocks after each MBConv for channel attention
  • Swish activation: x·σ(x) — smooth, differentiable, outperforms ReLU on deep nets
  • B0 baseline: 9 stage types, each stage repeated some number of times

ResNeXt
T-06 Xie et al., CVPR 2017
Aggregated Residual Transformations for Deep Neural Networks

Post-ResNet, the obvious scaling paths were deeper (more layers) or wider (more channels). Both work, but have diminishing returns and increase design complexity. ResNeXt introduces a third dimension: cardinality — the number of parallel transformation paths inside each residual block.

Empirical finding: Increasing cardinality from 1 to 32 (same total FLOPs) improves ImageNet accuracy more than doubling depth or width. Cardinality is a more effective scaling dimension than either depth or width.
  • Inception showed multiple parallel paths can be effective, but required hand-tuned topology per block
  • ResNeXt makes it uniform: every block has the same cardinality C, each path is identical
  • Grouped convolutions (used in AlexNet for multi-GPU) repurposed as an expressiveness tool
ResNeXt-50 32×4d: C=32 groups, d=4 channels wide per group Standard ResNet-50 bottleneck: 256ch → 64ch (1×1) → 64ch (3×3) → 256ch (1×1) ResNeXt-50 block (equivalent FLOPs): 256ch → 128ch (1×1) → 128ch grouped-3×3, C=32 → 256ch (1×1) Each group processes 4 channels independently Three equivalent formulations: (a) C separate branches: each 256 → 4 → 4 → 256, results summed (b) Concat: group outputs concatenated, then 1×1 combined (c) Grouped conv: single op with groups=C ← most efficient in practice
  • Aggregated transformations: output is the sum of C parallel transformations (not concat)
  • Simple rule: keep total FLOPs equal to ResNet baseline — just distribute them across C groups
  • Grouped conv in PyTorch: nn.Conv2d(in, out, 3, groups=C)

ShuffleNet
T-07 Zhang et al., CVPR 2018
Channel Shuffle for Group Convolution

MobileNet's depthwise separable convolutions are efficient, but the 1×1 pointwise convolution still dominates the computation budget at small channel counts. Replacing 1×1 with grouped convolution saves FLOPs, but creates a silo problem: groups cannot see each other's features.

  • In grouped conv, group 1's output only depends on group 1's input
  • Stacking grouped convolutions: information stays permanently siloed in its original group
  • The network loses the ability to learn cross-group feature combinations
ShuffleNet's solution: Channel Shuffle — a zero-parameter operation that redistributes channels between groups after each grouped conv. Now every group's input contains one channel from every original group, restoring full cross-group mixing.
Channel Shuffle (g groups, n total channels): 1. Reshape: (N, C, H, W) → (N, g, C/g, H, W) 2. Transpose: (N, g, C/g, H, W) → (N, C/g, g, H, W) 3. Flatten: (N, C/g, g, H, W) → (N, C, H, W) Effect: channel i in group j maps to channel j in group i → Each output position receives 1 channel from each original group ShuffleNet Unit: Input → GConv 1×1 → Channel Shuffle → DWConv 3×3 → GConv 1×1 → Add skip
  • Groups g=3 or 8 commonly used — larger g means each group is narrower (cheaper)
  • Downsampling unit: stride-2 DWConv + avg-pool on skip → concatenate instead of add (wider output)
  • ShuffleNet v2 (2018): adds channel split before + concat after — eliminates element-wise add overhead

Xception
T-08 Chollet, CVPR 2017
Extreme Inception — depthwise separable as the logical limit of Inception

The Inception architecture used multiple parallel branches (1×1, 3×3, 5×5 filters) to capture patterns at different scales. Each branch still mixed channel and spatial dimensions within it. Chollet's question: what if we push this decomposition to its logical extreme?

Inception hypothesis: Cross-channel correlations and spatial correlations can be mapped completely separately. A standard Inception module partially decouples them. Xception (Extreme Inception) fully decouples: one operation purely spatial, one purely cross-channel.

The result is mathematically equivalent to a depthwise separable convolution — but arrived at from a completely different theoretical direction than MobileNet.

Standard Inception branch: conv (mixes channels AND spatial simultaneously) Xception path: 1×1 conv (cross-channel only, no spatial mixing) → 3×3 depthwise (spatial only, per-channel, no cross-channel) vs MobileNet order: 3×3 depthwise → 1×1 pointwise (spatial first, then channel) Xception: 1×1 pointwise → 3×3 depthwise (channel first, then spatial) Key difference from MobileNet: No ReLU between pointwise and depthwise (empirically better for Xception)

Architecture Flow

  • Entry flow: 4 depthwise sep blocks, channels 64→128→256→728 with residual connections
  • Middle flow: 8× identical 728-channel residual blocks (no downsampling)
  • Exit flow: 1024 → sep(1536) → sep(2048) → global avg pool → dense → softmax
  • Total: 36 depthwise separable conv layers, all with residual connections

NASNet
T-09 Zoph et al., CVPR 2018
Neural Architecture Search — machines design machines

All architectures so far — ResNet, MobileNet, DenseNet — were designed manually. An expert would propose a structure, train it, measure accuracy, and iterate. This process takes months and depends heavily on intuition. Neural Architecture Search asks: can we automate this with machine learning itself?

  • Search space: even with 10 layer types and 10 layers, there are 10¹⁰ possible architectures
  • Exhaustive search is completely infeasible — even one training run takes hours
  • NAS trains a controller RNN that generates architecture descriptions
  • The controller learns via Reinforcement Learning: good architectures (high val accuracy) get positive reward
NASNet innovation: Don't search full architectures. Search for a repeatable cell (Normal Cell + Reduction Cell), then stack it. This reduces the search space dramatically and enables transfer from cheap CIFAR-10 search to expensive ImageNet deployment.
Controller RNN generates a cell by predicting sequences: For each of 5 blocks in a cell: Hidden state 1: one of {prev outputs, skip connections} Operation 1: one of {sep 3×3, sep 5×5, avg pool 3×3, max pool 3×3, identity, dilated 3×3, 1×7 then 7×1, sep 7×7, ...} Hidden state 2: same choices Operation 2: same op choices Combine: add or concat Total: 5^(4×ops + 2×inputs) possibilities per cell Controller trained with REINFORCE on CIFAR-10 val accuracy

Deployment

  • Search ran on 500 GPUs for 4 days (≈2000 GPU-days) on CIFAR-10
  • Best Normal Cell contains: sep 5×5, identity, avg pool 3×3, sep 3×3, max pool 3×3 operations
  • Stacking: stem conv → [N × Normal Cell] → [Reduction Cell] → repeat → classifier
  • N and initial channels are hyperparameters tuned to different compute budgets

Inverted Residuals and Linear Bottlenecks

MobileNetV1 worked, but applying ReLU inside the narrow bottleneck of a depthwise separable block caused information loss. Here's why: neural network feature maps typically lie on a low-dimensional manifold embedded in a high-dimensional space. ReLU is safe in high dimensions (zeroing some features doesn't collapse the manifold), but in low dimensions it crushes information irreversibly.

Linear manifold hypothesis: Interesting activations lie on a low-dimensional manifold. If we project to a low-dimensional space and apply ReLU, we collapse the manifold — information is permanently lost. The fix: no activation (linear) at the narrow projection step.
  • ResNet: wide → narrow → wide. Narrows to reduce computation, expands to restore capacity
  • But: ReLU in the narrow space hurts. MobileNetV1 had ReLU everywhere
  • MobileNetV2 inverts the residual: narrow → wide → narrow. The wide intermediate space can safely apply ReLU. The narrow output uses no activation.
MBConv Block (Inverted Residual + Linear Bottleneck): Input: n channels (narrow) → 1×1 expand: t×n channels + ReLU6 (expand ratio t, typically 6) → 3×3 depthwise: t×n channels + ReLU6 (spatial transform) → 1×1 project: n' channels NO activation ← linear bottleneck → skip add (if stride=1 AND n == n') Params per block: n × (t·n) + (t·n) × 9 + (t·n) × n' For t=6, n=32, n'=32: 32×192 + 192×9 + 192×32 = 6144 + 1728 + 6144 = 14,016

A ConvNet for the 2020s — modernising ResNet with transformer-inspired design

Vision Transformers (ViT, 2021) and Swin Transformer started outperforming CNNs on ImageNet at large scale. The community began questioning whether CNNs were architecturally limited. ConvNeXt's response: take a vanilla ResNet-50 and systematically modernise it with design choices borrowed from Transformers. How far can a pure ConvNet go?

  • Swin Transformer achieved 86.4% top-1 with a hierarchical Transformer — previously out of reach for CNNs
  • But was the gain from self-attention itself, or from other design choices (training recipe, activation, normalisation)?
  • ConvNeXt answers: mostly the other design choices
ConvNeXt roadmap: ResNet-50 (78.8%) → modern training recipe → stem patch → stage ratio → inverted bottleneck → 7×7 depthwise kernel → separate downsampling → LN instead of BN → GELU → fewer norms/acts → 82.0%. No attention, no tokens.

Squeeze-and-Excitation Networks — channel-wise attention

In a standard CNN, every channel in a feature map is treated with equal importance. But different channels respond to different semantic content — edges, textures, object parts, backgrounds. For a given input image, some channels are far more informative than others. Why not let the network learn which channels to emphasise?

  • Channel 42 in layer 3 might activate strongly for dog ears — very useful for a dog classifier
  • Channel 13 might respond to generic background texture — less useful for that specific input
  • Standard conv weights are fixed — same channel importance regardless of input content
  • SE asks: can we compute input-dependent channel weights cheaply?
Key insight: Global average pooling can summarise the "channel response strength" into a single number per channel. Two FC layers can then learn which channel combinations to boost and suppress for a given input — adaptively, at inference time.
SE Block: Squeeze: z_c = (1/H×W) Σ_{h,w} u_c(h,w) [global avg pool → C-dim vector] Excitation: s = σ( W₂ · δ( W₁ · z ) ) W₁: C → C/r (reduction, e.g. r=16) W₂: C/r → C (expansion) δ = ReLU, σ = Sigmoid [output: C weights ∈ [0,1]] Scale: x̃_c = s_c · u_c [channel-wise multiply] Parameters per SE block: 2 × C²/r For C=256, r=16: 2 × 256²/16 = 8,192 params (tiny overhead)

Integration

  • SE block inserted after each residual block — the block's output is recalibrated before being added to the skip
  • Works with any architecture: SE-ResNet, SE-Inception, SE-MobileNet, SE-EfficientNet
  • At inference: global pool is nearly free (already computed for classifier), FC layers run at 1×1 spatial — very fast
  • Reduction ratio r=16: standard choice. r=4 gives slightly higher accuracy at more parameters
